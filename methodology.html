02Stage 02 · Semantic Matching

Meaning, not keywords.

Traditional keyword search misses candidates who describe the same skill with different words. Embedding-based matching understands intent — so a "language prediction system" still matches an "NLP engineer" requirement.

The keyword problem
JD says:  "Machine Learning Engineer with NLP experience"
Keyword "AI developer building language prediction systems" 0%
Semantic "AI developer building language prediction systems" 87%
From text to vector
Input text "ML eng with NLP"
Encoder · all-MiniLM-L6-v2 384-dim vector
Embedding [0.23, 0.87, -0.12, …]
cos(θ) = A · B ‖A‖ × ‖B‖
Cosine similarity · score ranges 0.0 (unrelated) → 1.0 (identical meaning)
03Stage 03 · Behavioural Signals

Nineteen signals, normalised.

Activity, responsiveness and profile completeness are captured across 19 metrics per candidate. Each is MinMax-scaled to a 0–1 range so disparate units become directly comparable.

Signal Type Direction Weight
xscaled = x − xmin xmax − xmin →   invert (1 − xscaled) for "lower is better"
MinMax normalisation · all signals mapped to a 0.0 → 1.0 range
04Stage 04 · Score Fusion

Two dimensions, one score.

The semantic and behavioural scores are blended into a single fit score. Weights reflect which dimension matters more for genuine job fit.

Final Score = 0.7 × Semantic + 0.3 × Behavioural
Semantic 70%
Resume-to-job meaning match
Behavioural 30%
Platform engagement & activity

Why 70/30? Skills and experience — the semantic dimension — matter most for genuine job fit. But a candidate who is active and responsive is far more likely to actually engage, so behavioural signals earn a meaningful 30% weight rather than being ignored.
